## Problem

Without strict runtime validation, LLM-produced UI descriptors will cause brittle client errors.

## Proposed solution

Add `ui/src/contracts/genui.ts` defining:
- Base fields: `id`, `type`, `version`, `data`
- Discriminated union by `type`
- `safeParse()` wrappers that return typed results or validation errors.

## Acceptance criteria

- Schemas cover initial component set and export TS types via `z.infer<>`.
- Validation errors are serializable and can be shown in UI.

## Test plan

- Unit tests for schema parsing and invalid cases.
