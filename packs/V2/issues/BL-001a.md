# BL-001a — Define orchestrator state, TaskResult, and DataBrief schemas (backend)

- Type: **task**
- Milestone: `M1-W0`
- Repo alignment: **missing**
- Labels: `backend`, `contracts`
- Depends on: `BL-001`

## Problem

Planner/worker/synth nodes need a shared typed state and stable payload shapes for events and persistence.

## Proposed solution

Add Pydantic models for:
- `PlanTask` (id, title, description, status, assigned_tools, deps, citations_required)
- `PlanSet` (version, tasks[], created_at, updated_at)
- `TaskResult` (task_id, summary, evidence_refs[], raw_tool_artifacts[], confidence)
- `DataBrief` (key_facts[], assumptions[], risks[], open_questions[], citations[], entities[])

Store schemas in `src/intelli/contracts/` (new package) and ensure they are JSON-serializable for run_event payloads.

## Repo touchpoints

- `src/intelli/contracts/* (new)`
- `src/intelli/agents/graphs/research_assistant.py`

## Acceptance criteria

- Schemas exist with JSON schema export support (for TS typing later).
- TaskResult and DataBrief are serializable and versioned.

## Test plan

- Unit: schema validation roundtrip (dict → model → dict).
