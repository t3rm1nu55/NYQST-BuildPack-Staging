## Problem

Once patterns and dimensions exist, users need repeatable processes (templates) to run analyses consistently across corpora.

## Proposed solution

Implement process templates:
- A template defines: selected frameworks, dimensions, patterns, thresholds, deliverables.
- Templates can be run as workflows (compile to LangGraph) and emit standard outputs.
- UI: template builder with presets; run history and comparisons.

## Acceptance criteria

- User can create a template and run it on a corpus; outputs are consistent and versioned.

## Test plan

- Integration: template run generates expected artifacts.
